# 安全政策

## 支持版本

我们发布补丁，以解决以下版本中的安全漏洞：

| 版本 | 支持状态 |
|------|----------|
| 1.0.x | ✅ 支持 |
| 0.9.x | ❌ 不支持 |
| < 0.9 | ❌ 不支持 |

## 报告安全漏洞

如果您发现安全漏洞，请不要在公共问题跟踪器中报告。相反，请通过以下方式联系我们：

📧 **安全邮箱**: security@neuralfive.com

请在您的报告中包含以下信息：

- 漏洞的类型和影响
- 重现漏洞的步骤
- 可能的解决方案（如果已知）
- 您的联系信息（可选）

## 响应时间

我们承诺在以下时间内响应安全报告：

- **确认收到**: 24小时内
- **初步评估**: 3个工作日内
- **完整响应**: 14个工作日内

## 安全最佳实践

### 代码安全

- 所有代码都必须通过静态分析工具检查
- 敏感数据必须加密存储
- 用户输入必须进行验证和清理
- 使用安全的随机数生成器

### 依赖管理

- 定期更新依赖包
- 使用工具扫描已知漏洞
- 锁定依赖版本以确保可重现的构建

### 数据保护

- 不存储不必要的用户数据
- 使用适当的加密算法
- 实施数据访问控制

## 安全更新

当发现安全漏洞时，我们将：

1. 私下修复漏洞
2. 发布安全更新
3. 发布安全公告
4. 在24小时内通知已知受影响的用户
